命名規則　

メンバ数は先頭にm_をつけてアッパーキャメル
例
m_PlayerJump


ローカル変数はローワーキャメルで引数のかっこのvoidは書かない
例
int playerJump


関数はアッパーキャメル
例
int GetFlg()


forやifなどの鍵かっこは条件式の一段下にする(ifの処理が１行だけの時は鍵かっこは使わない）
例
for(n<10)
{
 //処理
}
ifのフラグ処理(true&false)などは!にする
例
if(!flg)
{
 //処理
}


クラスで一行で終わる処理の関数を作った場合鍵かっこはは一行下にかく
例
int GetFlg()
{
 return a;
}


flgの変数を作る際には語尾に○○Flgをつける
例
bool m_HogeFlg


変数の名前は英語でローマ字禁止


長い処理や複雑な処理にはコメントをつけてほしい
ファイル名はスネークケース
例
move_jump
